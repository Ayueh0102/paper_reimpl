# 04 — VQ-Font (AAAI 2023)

> Yao, Liu, Tang, "VQ-Font: Few-Shot Font Generation with Structure-Aware
> Enhancement and Quantization", AAAI 2023. Notes assembled blind from the
> Obsidian vault note `036_VQ-Font結構感知字體_AAAI2023.md` and the Phase 0
> spec row for paper 04. The official repository
> (`github.com/Yaomingshuai/VQ-Font`) was **not** consulted.

## TL;DR

VQ-Font casts few-shot Chinese font generation as a **token-prior refinement**
problem on top of a pre-trained VQGAN font codebook. The pipeline is
deliberately two-stage:

1. **Stage 0 — VQGAN font codebook pretrain.** A standard VQGAN
   (encoder → vector-quantizer → decoder) is trained end-to-end on a large
   high-quality multi-font corpus so the **codebook of size 1024** captures
   real-world glyph appearance priors (serif terminals, stroke connectivity,
   brush textures, art-font idiosyncrasies). Spatial grid is **16×16** for a
   128 px input — a factor-8 spatial downsample.
2. **Stage 1+ — Token prior refinement.** Given an *initial synthesized
   glyph* produced by any existing FFG synthesis module plus **3 reference
   glyphs**, a Transformer with cross-attention predicts which codebook index
   should live at each of the 16×16 feature positions. Decoding the predicted
   indices through the frozen VQGAN decoder yields the final glyph.

The third moving part is the **Structure-level Style Enhancement Module
(SSEM)**. The paper observes that prior work (e.g. FS-Font) does
patch-level style attention that ignores the natural compositional units of
Chinese characters. SSEM bakes the **12 Chinese-character structure classes**
(left-right, top-bottom, three-vertical, three-horizontal, full surround,
five different open-surround variants, overlap) into the Transformer's
attention so style features are matched **along structural boundaries**
(radicals / components) rather than uniform image patches.

## Architecture

```mermaid
graph TD
    subgraph Stage0[Stage 0 — VQGAN pretrain]
        x0[Target glyph x] --> E0[VQGAN encoder]
        E0 --> q0[VectorQuantize: codebook K=1024]
        q0 --> D0[VQGAN decoder]
        D0 --> rec[recon]
    end
    subgraph Stage1[Stage 1+ — Token Prior Refinement]
        s[Initial synthesized glyph] --> Eq[Encoder (frozen)]
        r1[ref 1] --> Er[Encoder (frozen)]
        r2[ref 2] --> Er
        r3[ref 3] --> Er
        sid[structure_id (0..13)] --> SE[SSEM StructureEncoder]
        Eq -->|"query tokens [B,N,C]"| TR[Transformer]
        Er -->|"context tokens [B,R*N,C]"| TR
        SE -->|"additive bias on queries + prefix token"| TR
        TR --> H[token head]
        TR --> SH[SSEM structure head]
        H -->|"argmax / sample"| idx[predicted indices]
        idx --> Dec[VQGAN decoder (frozen)]
        Dec --> out[final glyph]
    end
```

### Stage 0 — VQGAN font codebook

* **Encoder**: 3 down-stages (paper's 16×16 latent on 128 px ⇒ 8× downsample)
  built from `(ResBlock, ResBlock, Downsample)` triples (taming-transformers
  shape) terminating in a `Res–Attn–Res` mid-block and `GroupNorm + Conv` to
  produce `z_channels = 256` continuous features.
* **VectorQuantize**: Euclidean lookup into an `nn.Embedding(K=1024, D=256)`
  table with the standard VQ-VAE commitment + codebook loss
  `L_codebook = ‖sg[z_e] - z_q‖² + β · ‖z_e - sg[z_q]‖²` (β = 0.25, the SimCLR /
  Van den Oord default). Straight-through estimator carries gradient back
  through the discrete bottleneck.
* **Decoder**: mirror of the encoder using nearest-neighbour upsample, plus
  the same mid-block, plus a final `Conv` back to the input channel count.

This blind reimpl **omits the adversarial discriminator** that the original
VQGAN paper introduced. The note for VQ-Font says "VQGAN-based framework"
but does not specify discriminator architecture, GAN loss weight, or LeCAM /
spectral-norm details. We expose an extension hook (`recon_weight`,
`vq_weight` in the train YAML) and leave a `[guessed-because-paper-vague]`
note: a future Phase 2 diff against the official repo is expected to add the
discriminator.

### Stage 1+ — Token Prior Refinement Transformer

A Pre-LN Transformer (Pre-LN keeps training stable without warmup) with
`num_blocks = 6` blocks (depth not stated in the paper note — guessed) and
`num_heads = 8` (**paper-cited**). Each block is `Self-attn → Cross-attn →
MLP` with LayerNorm on every input. Cross-attention queries are the 16×16
encoder features of the **initial synthesized glyph**; keys/values are the
concatenation of the 3 reference glyphs' encoder features plus a single SSEM
prefix token. A learnable null-token is substituted when `ref_valid[i,r]` is
false so cross-attn keys never collapse to a fully-masked row.

The final head is a per-position `Linear(C, K=1024)` producing logits over
the codebook. Decoding via argmax (paper default) and via temperature/top-k
sampling are both supported.

### SSEM — Structure-level Style Enhancement Module

The 12-way Chinese structure prior is sourced from the mother repo's
`scripts/lookup_ids.py::parse_structure`. That function maps the leading IDS
character of a CJK character to one of:

```
left_right / top_bottom / left_mid_right / top_mid_bottom /
surround_full / surround_open_{bottom,top,right,TR,TL,BR} /
overlap / atomic
```

— exactly 12 named structures + `atomic` for non-decomposable single-glyph
characters. We add a 14th `unknown` sentinel so synthetic / TTF-pretrain
paths that do not yet have an IDS lookup still route through a valid class.
SSEM consists of:

1. `StructureEncoder` — an `nn.Embedding(14, embed_dim)` that produces a
   per-sample structure vector. This vector is added to every query token
   (FiLM-style additive bias) **and** prepended to the reference context as
   a global structure prefix so cross-attn can route to it.
2. `StructureHead` — a tiny `LayerNorm → Linear` head that re-classifies
   the structure id from the mean-pooled Transformer output, used as an
   **auxiliary cross-entropy loss** `L_struct` weighted by `λ_struct`. This
   is our operationalisation of the paper's "structure-level style matching";
   the paper does not give a loss form, so this is `[guessed]`.

## Loss equations (this reimpl's notation)

Stage 0 — VQGAN:
```
L_vqgan = L_recon (L1) + λ_vq · L_vq
        = mean | x̂ − x |₁  +  λ_vq · ( ‖sg[z_e] − z_q‖² + β · ‖z_e − sg[z_q]‖² )
```

Stage 1+ — Token Prior Refinement:
```
target_idx = VQGAN.encode_indices(x_target)       # frozen, no grad
logits     = Transformer(E(synth), E(refs), sid)  # [B, N, K]
L_token    = CE(logits, target_idx)               # per-token cross-entropy
L_struct   = CE(structure_head(h), sid)           # SSEM aux objective
L_total    = L_token + λ_struct · L_struct
```

Optimizer: AdamW(β₁=0.9, β₂=0.999), gradient clip 1.0. Learning rates:
**paper-cited** 4e-5 for Stage 0 (200k iters) and 2e-4 for Stage 1+
(300k iters), batch size 32 on 1× A6000.

## Data flow per step (Stage 1+)

1. Dataset emits `image` (target x in [-1, 1]), `content` (source-glyph
   render — Phase 1 stand-in for the initial-synthesis output), `ref_images`
   of shape `[B, R, C, H, W]` with `ref_valid` mask, and `structure_id`
   from the manifest (`row['structure_id']` if pre-computed by manifest
   build, else `parse_structure` on the row's char).
2. `VQFont.encode_target_indices(image)` queries the frozen VQGAN to produce
   the target `[B, H_lat, W_lat]` codebook indices. **No grad.**
3. `VQFont.predict_token_logits(initial, refs, structure_id, ref_valid)`
   runs `encoder(initial)` and `encoder(refs.flatten(B*R))` (both **no grad**
   on the frozen VQGAN), then the Transformer cross-attends and emits
   `(token_logits, structure_logits)`.
4. `compute_loss` computes per-token CE + structure-CE, returns
   `(loss_total, log_dict)`.
5. `optim.step()` updates **only the Transformer** (frozen VQGAN is filtered
   out by `requires_grad`).

## Conditioning paths (each must be testable)

- **Initial-synthesis path**: `initial → encoder → query tokens`. Smoke test
  asserts cross_attn parameters receive gradient and that the SSEM additive
  bias on queries is grad-connected.
- **Reference path**: `refs → encoder → context tokens`. Cross-attn K/V
  gradient is checked in the smoke test; padded slots use a learnable null
  token so a fully-padded batch still has finite softmax keys.
- **Structure path**: `structure_id → StructureEncoder embedding →
  additive bias on queries + prefix token in context`. SSEM encoder
  gradient is explicitly checked.
- **Frozen VQGAN**: smoke test confirms every parameter has `requires_grad =
  False` after `freeze_vqgan=True` and that no gradient accumulates after
  the backward pass.

## What this implementation does NOT match (yet)

- **VQGAN adversarial loss** is omitted. Paper says "VQGAN-based framework"
  but does not specify discriminator architecture or loss weight. Phase 2
  diff is expected to surface this.
- **External "synthesis module"**: the paper's Stage 1 input is the output
  of any existing FFG method (e.g. DG-Font, FS-Font). Phase 1 uses the
  source-glyph render itself as the stand-in. Plumbing a real synthesis-module
  checkpoint into `compute_loss` is left as a Stage B/C exercise.
- **SSEM loss form**: the paper describes SSEM in prose ("structure-level
  style matching") but does not give equations. We operationalise it as a
  14-way CE on the structure id. A contrastive variant (same-structure
  positive, cross-structure negative) is a possible Phase 2 swap.
- **Exact codebook size for the codebook** is 1024 from the paper; we keep
  it parametric (`vqgan.num_embeddings`) so future ablations are cheap.
