# 06 — Calliffusion (Liao 2023, arXiv 2305.19124)

Blind reimplementation notes for paper 06_calliffusion. Compiled from the
Obsidian paper note `Liao_2023_arXiv_Calliffusion_CalligraphyGenerationStyleTransfer.md`
plus row 06 of `reports/phase0_spec_table.md` and the arxiv abstract page.
**No official code was consulted.** Anything not explicitly attributable to the
paper note carries a `[guessed-…]` tag in `reports/blind_impl.md`.

## 1. One-line description

A conditional DDPM that generates 5-script Chinese calligraphy images
conditioned on a natural-language text prompt of the form
`<character> <script> <calligrapher>` (e.g. `人字 隸書 曹全碑`). The prompt is
embedded by a frozen Chinese BERT (768 dim) and injected into the U-Net via
cross-attention at every resolution stage. One-shot style transfer to an
unseen character or alphabet (English letters / digits) is done by adding
LoRA adapters to the cross-attention projections and fine-tuning on a single
reference image.

## 2. Architecture

```mermaid
flowchart LR
  P[Text prompt<br/>"人 隸書 曹全碑"] --> TK[Chinese BERT tokenizer]
  TK --> BERT[Chinese BERT 768-dim<br/>frozen during Stage A/B]
  BERT --> CTX[seq_len × 768 context]

  X[noisy image x_t<br/>1×H×W] --> CONV[conv-in 320 ch]
  T[timestep t] --> TE[sin/cos PE → MLP] --> TEMB[time emb 1280]

  CONV --> D1[Down block 320]
  D1 --> D2[Down block 640]
  D2 --> D3[Down block 1280]
  D3 --> D4[Down block 1280]
  D4 --> MID[Mid block 1280]
  MID --> U4[Up block 1280]
  U4 --> U3[Up block 1280]
  U3 --> U2[Up block 640]
  U2 --> U1[Up block 320]
  U1 --> CONVOUT[conv-out 1 ch] --> NPR[ε̂ predicted]

  CTX -- cross-attn k,v --> D1
  CTX -- cross-attn k,v --> D2
  CTX -- cross-attn k,v --> D3
  CTX -- cross-attn k,v --> D4
  CTX -- cross-attn k,v --> MID
  CTX -- cross-attn k,v --> U4
  CTX -- cross-attn k,v --> U3
  CTX -- cross-attn k,v --> U2
  CTX -- cross-attn k,v --> U1
  TEMB --> D1 & D2 & D3 & D4 & MID & U4 & U3 & U2 & U1
```

### 2.1 U-Net dimensions (from paper note p.3)
Block channel widths: `[320, 640, 1280, 1280]`. We interpret this as four
down-sampling stages (factor 2 each) plus a mid block plus four
up-sampling stages with skip connections. Within each stage we put
`num_res_blocks` ResNet blocks followed by one spatial self-attention block
and one cross-attention block that consumes the BERT context. For the smoke
build we use a configurable `image_size` (default 64 in tests, 128/256 in
real configs) and `num_res_blocks=2`.

### 2.2 Time embedding
Standard DDPM: sinusoidal position embedding of dim 320 → 2-layer MLP →
1280-dim time embedding broadcast as additive bias inside every ResBlock.

### 2.3 Cross-attention block
For each spatial token at resolution `H×W` and channel `C`, the block does
`Q = W_q · h, K = W_k · ctx, V = W_v · ctx` (multi-head, 8 heads in our
default). Output is added back to `h` via a residual. `ctx` is the BERT
last-hidden-state of shape `[batch, seq_len, 768]`, projected to `C` via a
linear `W_ctx`.

### 2.4 Chinese BERT integration
- We use `bert-base-chinese` (HuggingFace model id) as the default
  pre-trained tokenizer + encoder. Hidden size 768.
- Tokenizer is fed a single string of the form
  `f"{character} {script} {calligrapher}"` (script and calligrapher names
  are Chinese strings).
- **Vocabulary gap**: BERT-base-chinese's tokenizer is character-level for
  CJK but unknown calligraphers (e.g. `顏真卿`, `王羲之`, individual rare
  characters) may decompose into per-character pieces. The original paper
  is silent on whether new tokens were registered. Our blind decision is
  to **add the 24 二南堂 calligrapher names as `additional_special_tokens`
  at tokenizer build time and `resize_token_embeddings` on the BERT model
  so the new embedding rows can be trained**. This is consistent with the
  paper's claim that style names are recognised as semantic units rather
  than splintered subwords. See `reports/blind_impl.md` for full reasoning.
- BERT is **frozen** during Stage A (TTF pretrain) and Stage B
  (multi-writer mid-train), then **unfrozen for the writer-name embeddings
  only** during Stage C (二南堂 fine-tune) so the model can learn the
  per-writer style anchor. The U-Net cross-attention `W_k / W_v` are
  trainable throughout.

### 2.5 LoRA adapters
At Stage C and at one-shot style-transfer time we attach LoRA adapters to
the **cross-attention projection matrices** `W_q / W_k / W_v / W_out` in
every U-Net cross-attention block. Rank `r=4`, scaling `α=8`. Base
parameters are frozen; only LoRA `A`, `B` matrices are trained. A
minimal in-house LoRA wrapper (see `src/calliffusion/lora.py`) is used so
we avoid taking a hard dependency on `peft` for the smoke test.

## 3. Loss

Pure DDPM ε-prediction loss:

```
L_simple(θ) = E_{x0, t, ε} [ || ε - ε_θ(x_t, t, c_BERT) ||^2_2 ]
```

where `x_t = sqrt(ᾱ_t) · x0 + sqrt(1 - ᾱ_t) · ε` and `c_BERT` is the
BERT-encoded prompt. We use `reduction='mean'`. No auxiliary VLB term, no
LPIPS, no perceptual. The paper note explicitly says "MSE diffusion loss
only" so this is fixed.

For LoRA fine-tuning the same loss is used; only the active parameter set
shrinks.

## 4. Data flow per training step

1. Sample a row from the manifest: `(image_path, char, script, writer)`.
2. Load the calligraphy image, convert to grayscale `[1, H, W]` in `[-1, 1]`.
3. Build prompt string `f"{char} {script} {writer}"`.
4. Tokenize with Chinese BERT tokenizer → `input_ids, attention_mask`.
5. Run BERT → `last_hidden_state` of shape `[B, L, 768]`.
   - During training: `with torch.no_grad()` when BERT is frozen.
6. Sample `t ~ Uniform{0..T-1}`, `ε ~ N(0, I)`, build `x_t`.
7. Forward `ε_θ(x_t, t, ctx=BERT_out)` → predicted noise `ε̂`.
8. `loss = MSE(ε, ε̂)`; backward; optimizer step.

## 5. Schedules

- **Noise schedule** (paper note p.3, training-config section): linear
  variance `β_1 → β_N`. We default `β_1 = 1e-4, β_N = 0.02, T = 1000`.
- **Sampler at inference**: DDPM (paper note). We provide both DDPM and a
  DDIM variant from `paper_reimpl_shared.diffusion.gaussian` because the
  shared `GaussianDiffusion` already supports both and exposing DDIM costs
  us nothing.
- **Prediction target**: ε-prediction (standard DDPM). Note the shared
  helper also supports `x0` prediction; we override to `epsilon` because
  paper note shows a vanilla DDPM and never mentions x0/v parameterisation.

## 6. Training schedule

Original paper: batch 16, lr 1e-5, 2× A100 40GB × 120 h (~24M images
processed at roughly 50–60 it/s effective). Our scale-down for single-GPU
repro on Ernantang follows the 3-stage convention used elsewhere in the
repo:

- **Stage A — TTF pretrain**: ~13 fonts × 4659 chars rendered to images.
  Prompts use `<char> 楷書 <font_name>`. Batch 16, lr 1e-5, 100k steps.
  BERT frozen. Goal: learn the generic glyph manifold conditioned on text.
- **Stage B — multi-writer mid-train**: scaled-down calligraphy corpus
  (Ernantang 24 writers + any synthetic prompts). Batch 8, lr 5e-6, 50k
  steps. BERT frozen except for added writer-name token embeddings.
- **Stage C — 二南堂 fine-tune via LoRA**: attach LoRA to cross-attention
  projections, freeze everything else. Batch 4, lr 1e-4 for LoRA params
  (LoRA can take a higher LR than the base model since `B` is initialised
  to zero). 10–20k steps. Optionally one-shot per writer/script slice.

`condition_dropout` for CFG: paper does not mention CFG explicitly, but
empirically diffusion-from-text always uses CFG at inference. We set
`prompt_dropout_p = 0.1` (drop the entire prompt → empty string) during
training so we can do CFG at sample time. **[guessed-because-paper-vague]**

## 7. Conditioning paths

| Conditioning signal | Path |
|---|---|
| character identity | BERT prompt → cross-attn (only path) |
| script identity | BERT prompt → cross-attn |
| calligrapher identity | BERT prompt → cross-attn; added special token |
| time step `t` | sin/cos → MLP → additive bias in every ResBlock |
| one-shot style sample | LoRA adapters on cross-attn `W_q/W_k/W_v/W_out` |

There is **no** categorical writer-id embedding table à la Moyun's
TripleLabel: all style information goes through the natural-language text
path. This is the paper's key design difference vs Moyun.

## 8. Data normalisation

- Image: grayscale, contain-fit to `H×H` with white padding, scale to
  `[-1, 1]`. Same convention as the shared `CalligraphyJsonlDataset`.
- Prompt: raw UTF-8 string; the BERT tokenizer handles normalisation.
- No augmentation that flips or rotates (calligraphy is orientation-aware).
  Mild scale/translate jitter is acceptable; off by default in our configs.

## 9. Compatibility with Ernantang

Strong match on metadata: Ernantang ships `(writer, work/inscription,
script, char)` per glyph which trivially serialises to the paper's prompt
template. The big gap is **scale**: paper trained on 1387 calligraphers ×
2025 chars (~half a million images). Ernantang has 24 writers × 4659 chars,
so we expect a smaller per-writer effective sample. We rely on Stage A TTF
pretraining to absorb glyph-shape priors before Stage B sees writer style.

For the BERT vocabulary, the 24 Ernantang writer names (parsed from the
manifest's `writer` field) are added as additional special tokens at
dataset-construction time. Stage B fine-tunes the embedding rows for these
24 new tokens together with the U-Net.

## 10. Risks (carried into blind_impl.md)

- BERT may produce vacuous semantics on Chinese calligrapher names that
  weren't in its training set; paper itself notes (citing Moyun) that BERT
  is weak on these. Mitigation: special-token + embedding fine-tune.
- Scale-down to 24 writers may underfit; expect lower writer-conditional
  fidelity than the paper.
- Without the original BERT freeze schedule we have to guess when to unlock
  embeddings vs full BERT vs full model. We default to embeddings-only at
  Stage C.
- LoRA rank `r=4` is taken from common SD-LoRA practice; paper does not
  state a number.
- CFG dropout rate `p=0.1` is guessed (paper silent).
- 2×A100×120h is far beyond our single 6000-Ada budget; Stage A is the
  bottleneck — we'll likely shorten to 50k steps and accept reduced glyph
  prior quality.
