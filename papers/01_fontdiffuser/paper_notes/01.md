# 01 — FontDiffuser (AAAI 2024)

> Yang et al., "FontDiffuser: One-Shot Font Generation via Denoising Diffusion
> with Multi-Scale Content Aggregation and Style Contrastive Refinement",
> AAAI 2024. Notes assembled blind from the Obsidian vault note
> `020_FontDiffuser單樣本擴散字體_AAAI2024.md` and Phase 0 spec row 01. The
> official repository was not consulted.

## TL;DR

FontDiffuser is the first paper to formulate one-shot Chinese/CJK font
generation as a **noise-to-denoise** problem with a U-Net DDPM. Conditioning is
purely image-based: a single source-glyph render (content) plus a single
style-reference glyph. The paper's three contributions are wired into the
denoiser as three named blocks:

1. **MCA — Multi-scale Content Aggregation** fuses content-encoder features
   into the U-Net at multiple resolutions, so fine stroke structure survives
   the "hard-complexity" character regime that GAN baselines (DG-Font /
   MX-Font / CF-Font) drop strokes on.
2. **RSI — Reference-Structure Interaction** is a cross-attention block that
   lets U-Net tokens attend to the style reference, modelling geometric /
   structural deformation between the reference and the target character.
3. **SCR — Style Contrastive Refinement** is a separate style-extractor
   network supervised by a style-contrastive loss (same-style positive,
   different-style negative). It is used as an auxiliary objective on the
   diffusion model's predicted x₀.

All other moving parts (β schedule, T, optimizer, exact channel widths)
are not specified in the paper note or Phase 0 row. They are filled with
DDPM-standard defaults and documented as guesses in `reports/blind_impl.md`.

## Architecture

```mermaid
graph TD
    src[Source glyph<br/>(rendered TTF, content_channels)] --> CE[Content Encoder]
    ref[Style reference<br/>(1 grayscale glyph)] --> SE[Style Encoder]
    CE -->|"multi-scale feats (one per U-Net stage)"| MCA[MCA fusers]
    SE -->|"L style tokens [B,L,D]"| RSI[RSI cross-attn]
    xt[x_t noisy image] --> U[U-Net denoiser]
    t[timestep] -->|"sin/cos + MLP"| U
    MCA -->|"gated add at each stage"| U
    RSI -->|"cross-attn at attn_resolutions"| U
    U --> eps_hat[predicted ε  (or x_0)]
    eps_hat -.->|"SCR loss path"| SCR[StyleExtractor frozen]
    x0[true x_0] -.->|"contrast anchor"| SCR
    SCR --> Lscr[L_scr (contrastive)]
    eps_hat --> Lsimple[L_simple (MSE)]
    Lsimple --> Total
    Lscr --> Total[L = L_simple + λ_scr · L_scr]
```

### Content Encoder

Mirrors the U-Net's channel-mult / stage layout so its per-stage output drops
right into the matching U-Net stage. Stem conv at full resolution, then one
strided 3×3 conv block per stage (channels = base · mult). The decision to
mirror is forced by MCA: feeding stage-i U-Net features the content map at
stage-i requires identical spatial size and same channel width.

### Style Encoder

Shallow 4-layer strided CNN that downsamples the reference glyph 8× and flattens
the result into a token sequence of length L = (H/8) · (W/8). The trunk depth
is not stated in the note and we chose a shallow encoder; for 128 px input
that yields L = 16 · 16 = 256 tokens. Tokens are L2-normalized inside RSI by
the LayerNorm on K/V projections.

A learnable null-style token replaces the encoder output when (a) no reference
is provided (e.g. unconditional Stage A) or (b) the per-sample CFG mask drops
the reference. This keeps the cross-attention block well-defined under
classifier-free guidance.

### U-Net Denoiser

Standard DDPM-shaped U-Net: input conv → down stages (each: N res blocks →
optional MCA fuse → optional self-attn → optional RSI cross-attn → downsample)
→ middle (res, self-attn, RSI, res) → up stages (skip concat → N+1 res blocks
→ MCA → self-attn + RSI at flagged resolutions → upsample) → output norm +
conv. Time embedding is sin/cos (Ho 2020) followed by a 2-layer MLP, FiLM-style
modulation injected into every res block via per-channel additive bias.

> [REVISED PER PHASE 2 — 2026-05-11] **RSI placement**: the deformable-conv
> RSI fires only on the up path at the inner stages (default
> `rsi_up_stages=(1, 2)` for a 4-stage U-Net), mirroring the official
> `('UpBlock2D','StyleRSIUpBlock2D','StyleRSIUpBlock2D','UpBlock2D')`
> layout at `third_party/01_fontdiffuser/src/build.py:19-22`. **MCA
> placement** is symmetric: inner stages only. Style cross-attention (the
> lighter RSIBlock that consumes style tokens) fires alongside MCA at
> every MCA stage on both paths, plus the middle block.

> [REVISED PER PHASE 2 — 2026-05-11] **RSI operator**: replaced the plain
> cross-attention `RSIBlock` (which we still keep for the style token
> branch on the down path) with a deformable-conv warp on the *skip*
> connection. See `### RSI (deformable-conv warp)` below.

Self-attention fires at every resolution listed in `attn_resolutions`.
Default places it at 16×16 only (the U-Net bottleneck for 128 px input),
which is the same convention as Improved-DDPM / OpenAI guided-diffusion.
The middle block always carries both self-attn and style cross-attn (the
token-level RSI).

### RSI (deformable-conv warp) — Phase 2

> [REVISED PER PHASE 2 — 2026-05-11] Added a new module
> `OffsetRefStrucInter` that re-implements the paper's actual RSI: a
> learned spatial warp of the U-Net skip connection, parameterised by an
> offset map predicted from `(skip, style_content_feat)`.

Pipeline at each up-path stage in `rsi_up_stages`:

1. The skip feature from the matching down stage (after the res blocks)
   is the input.
2. The matching-resolution feature from `ContentEncoder(style_image)`
   (the **style-content pyramid**) is the conditioning context.
3. Both are GroupNorm + SiLU normalised, 1×1-projected to a shared inner
   dim, and combined through a small position-wise MLP.
4. A 1×1 conv head emits the DCN offset map (`2 * kH * kW` channels for
   a `(kH, kW)` kernel; default 3×3 → 18 channels).
5. `torchvision.ops.DeformConv2d` warps the skip in place using the
   predicted offsets.
6. `mean(|offset|)` is accumulated as `offset_l1` and the trainer adds
   `offset_l1_weight * sum(offset_l1)` to the loss (default 0.5).

Concept origin:
`third_party/01_fontdiffuser/src/modules/attention.py:266-332`
(offset prediction) and
`third_party/01_fontdiffuser/src/modules/unet_blocks.py:423-587`
(`StyleRSIUpBlock2D` warp + offset accumulation).

The style-content pyramid is produced by a **second** `ContentEncoder`
instance — `model.style_content_encoder` — fed with `ref_images[:, 0]`.
The official repo parameter-ties one content encoder across both
`content` and `style` inputs (`third_party/01_fontdiffuser/src/model.py:43-44`).
We separate the two instances so the gradient paths are independently
testable, at the cost of duplicating the encoder weights.

### MCA fuser

> [REVISED PER PHASE 2 — 2026-05-11] Initial blind impl was a zero-init
> gated additive blend. Replaced with the official "concat + 1×1 Conv +
> Squeeze-Excitation + 1×1 Conv" channel-attention block, and gated to
> fire only at the **inner two stages** of the 4-stage U-Net (matching
> `('DownBlock2D','MCADownBlock2D','MCADownBlock2D','DownBlock2D')`
> placement at `third_party/01_fontdiffuser/src/build.py:15-22`). See
> `OURS::src/fontdiffuser/model.py::MCAFuse` and
> `FontDiffuserConfig.resolved_mca_stages()`.

Operation per MCA stage:

1. Project the matching-resolution content-encoder feature map to the
   U-Net stage's channel width with a 1×1 conv (and bilinear-resize if
   spatial dims drift).
2. Concat the projected content with the U-Net feature on the channel
   dim. Width becomes `2 * unet_channels`.
3. GroupNorm + SiLU + 1×1 conv to mix channels.
4. Squeeze-Excitation channel-attention with default `reduction=32`,
   then a residual back onto the concat tensor.
5. GroupNorm + SiLU + 1×1 conv to project back to `unet_channels`.

Concept origin: `third_party/01_fontdiffuser/src/modules/attention.py:359-414`
(`ChannelAttnBlock`).

Old behaviour (kept for the diff record):

> "Project content feature to U-Net stage's channel width (1×1 conv),
> compute a per-channel gate (1×1 conv → sigmoid), gated-add: `h ← h +
> sigmoid(W_g · proj(c)) · proj(c)`. Zero-init gate."

That blend was inserted at *every* U-Net stage. The Phase 2 revision
restricts it to the inner stages, leaving the outer down/up stages as
plain `DownBlock2D`/`UpBlock2D` — this changes representational
bandwidth and channel budget, and is necessary to match any
"FontDiffuser-compatible" claim.

### SCR loss

> [REVISED PER PHASE 2 — 2026-05-11] Initial blind impl used a shallow
> 3-conv extractor with supervised NT-Xent over writer_id positives. That
> is *not* the paper's SCR. Replaced with the official InfoNCE formulation:
> pretrained VGG-16 trunk + 6 per-stage projector heads + InfoNCE
> (τ=0.07, `negative_mode='paired'`) + `kornia.RandomResizedCrop`
> positives + dataset-mined `num_neg=16` other-style same-content
> negatives. See `OURS::src/fontdiffuser/model.py::SCRModule`.

Architecture:

1. **VGG-16 feature trunk** (`torchvision.models.vgg16(pretrained=True)`)
   split at the 6 MaxPool boundaries into stages with out-channels
   `[64, 128, 256, 512, 512, 512]`. Each stage feature is GAP+GMP pooled
   and compressed through a 1×1 conv back to the per-stage width.
2. **6 projector heads**, each `[stage_C → 1024 → 2048 → 2048]` with L2
   normalisation. Heads are L2-normalised before InfoNCE.
3. **Loss**: `InfoNCE(temperature=0.07, negative_mode='paired')` summed
   across the requested `nce_layers` (default `(0,1,2,3)`), averaged.
4. **Positives**: `kornia.augmentation.RandomResizedCrop(scale=(0.8,1.0),
   ratio=(0.75,1.33))` applied to the **same** target image — positives
   are same-style/different-patch rather than identical pixels.
5. **Negatives**: `[B, num_neg, C, H, W]` other-style same-content
   images, supplied by the dataset layer as `batch["neg_images"]`.

Concept origin: `third_party/01_fontdiffuser/src/modules/scr.py:9-96` and
`scr_modules.py:5-105`.

The full pipeline assumes the SCR module itself has been pretrained
separately for ~210k steps and loaded via `scr_ckpt_path` (see
`third_party/01_fontdiffuser/scripts/train_phase_2.sh:9`). Stage A's
`scr_weight` is 0 so the SCR is dormant; Stage B/C ramp it up.

## Data flow per step

1. Dataset emits `image` (target x₀ in [-1, 1]), `content` (source glyph
   render), `refs` (one-shot style ref, [B, 1, C, H, W]), writer_id /
   style_family_id labels, and a `ref_valid` mask.
2. `GaussianDiffusion.sample_training_batch(x₀)` samples t ~ U(0, T-1) and
   produces (x_t, target, noise, t).
3. `FontDiffuser.forward(x_t, t, content=..., ref_images=..., ref_valid=...)`
   runs content encoder → style encoder → U-Net with MCA + RSI fusion → ε̂.
4. `L_simple = mse(ε̂, target)`.
5. If SCR is active: `x̂₀ = predict_x0(x_t, t, ε̂)`; embeddings via frozen
   StyleExtractor; supervised contrastive on writer_id. `L_total = L_simple +
   λ_scr · L_scr`.
6. Backprop into model weights only. Style extractor is frozen.
7. Sampling at eval: shared `GaussianDiffusion.sample(...)` with sampler =
   "ddpm" (T-step) or "ddim" (k-step). CFG between ref-conditioned and
   null-ref-unconditional paths.

### Loss recipe — Phase 2

> [REVISED PER PHASE 2 — 2026-05-11] Phase 1 loss is now three terms
> (mse + 0.01·perceptual + 0.5·offset); Phase 2 adds InfoNCE SCR on top.

```
L_phase1 = mse(eps_hat, eps)
         + perceptual_weight  * VGG16_Perceptual(x0_pred, x0_true)
         + offset_l1_weight   * sum_of_up_path_offset_L1
L_phase2 = L_phase1 + scr_weight * InfoNCE(x0_pred, target_x0, neg_images)
```

Coefficients per official `configs/fontdiffuser.py:44-50`:

| term            | coefficient | source on the official side                                          |
|-----------------|-------------|-----------------------------------------------------------------------|
| MSE on epsilon  | 1.0         | `train.py:195` (`F.mse_loss(noise_pred, noise, reduction='mean')`)   |
| Perceptual VGG  | 0.01        | `perceptual_coefficient`                                              |
| Offset L1 sum   | 0.5         | `offset_coefficient`                                                  |
| SCR InfoNCE     | 0.01        | `sc_coefficient` (Phase 2 only)                                       |

### Perceptual loss — Phase 2

> [REVISED PER PHASE 2 — 2026-05-11] Added a `ContentPerceptualLoss`
> module using torchvision `vgg16(pretrained=True)` slices `enc_1`/`enc_2`/
> `enc_3` (relu1_2, relu2_2, relu3_3). Inputs are re-normalised from the
> training `[-1, 1]` space to ImageNet stats. Grayscale inputs are
> repeated to 3 channels. Loss is mean MSE across the three feature maps.

Concept origin: `third_party/01_fontdiffuser/src/criterion.py:6-44`.

### CFG dropout — Phase 2

> [REVISED PER PHASE 2 — 2026-05-11] CFG dropout now drops **both**
> content and style for the same batch rows (probability `cfg_drop_prob`,
> default 0.1). Concept origin:
> `third_party/01_fontdiffuser/train.py:182-186`. Sample-time uncond
> uses `cfg_uncond_drops_content=True` (`OURS::src/fontdiffuser/sample.py`)
> to match the convention.

The earlier blind impl only dropped the style ref (`ref_valid[:, 0] =
False`) while keeping `content` intact. That asymmetry made
`cfg_scale > 1.0` pull toward OOD predictions — the fix is to align
the train-time and sample-time dropout protocols.

## Loss equations (this reimpl's notation)

Forward process (DDPM):
```
x_t = sqrt(α̅_t) · x_0 + sqrt(1 − α̅_t) · ε,   ε ~ N(0, I)
```

Simple loss (we default to ε-prediction):
```
L_simple(θ) = E_{t, x_0, ε} ‖ε_θ(x_t, t; src, ref) − ε‖²
```

Predicted x₀ (for SCR target):
```
x̂_0(x_t, t; ε̂) = (x_t − sqrt(1 − α̅_t) · ε̂) / sqrt(α̅_t)
```

SCR contrastive (writer_id positives within batch B):
```
sim_{ij} = ⟨φ(x̂_0)[i], stop_grad(φ(x_0))[j]⟩ / τ
L_scr = −(1/B) Σ_i (1/|P_i|) Σ_{j ∈ P_i} log( exp(sim_{ij}) / Σ_k exp(sim_{ik}) )
```

Total:
```
L = L_simple + λ_scr · L_scr
```

Classifier-free guidance dropout: with probability `cfg_drop_prob` (default
0.1) we zero out `ref_valid`, which routes the style branch through the
learned null-style token and trains the unconditional path.

## Conditioning paths (each must be testable)

- **Content path**: source glyph → content encoder → per-stage MCA fuse → U-Net.
  Smoke test asserts `content_encoder` parameters receive non-zero gradient.
- **Style path**: ref glyph → style encoder → token sequence → RSI cross-attn
  in U-Net. Smoke test asserts `style_encoder` parameters receive non-zero
  gradient. With `ref_valid=False` the path falls back to the learned null
  token (still differentiable, but the encoder receives no gradient that
  iteration — by design, since the encoder did not run).
- **Time path**: timestep → sin/cos → MLP → FiLM-add in every res block.
  Always active.
- **SCR path**: predicted x₀ → frozen extractor → contrastive against true x₀
  via writer_id labels. Smoke test #2 asserts SCR loss is positive and that
  the UNet still receives gradient (i.e. SCR does not detach the main path).

## What this implementation does NOT match (yet)

- **SCR negative mining**: paper note hints at "same-char-different-style"
  negatives. We use within-batch writer_id positives instead, which requires
  no structured batch sampler.
- **Style extractor pretraining**: paper expects the SCR extractor to be
  pretrained on writer-id classification. We provide the architecture
  (`StyleExtractor`) and freeze it during diffusion training, but the
  pretraining recipe is left out of Phase 1 (added in Phase 3 if needed).
- **Cross-language eval**: paper evaluates Chinese → Korean transfer. Our
  Stage configs only point at the Ernantang manifest set, which is
  Chinese-only — cross-language is deferred to Phase 3.
- **Exact channel widths / depths**: paper note does not state them. Our
  defaults (`base_channels=64`, `channel_mult=(1,2,4,4)`, `num_res_blocks=2`)
  are conservative and need tuning post-review.
