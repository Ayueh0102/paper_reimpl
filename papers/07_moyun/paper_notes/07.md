## 1. What problem does Moyun solve?

Three weaknesses motivate this paper:

1. **U-Net's local receptive field misses long-range stroke dependencies.**
   Calligraphy contains effects like 飛白 (flying-white) and 留白 (negative space) that span across the entire glyph; a small conv kernel can't see them. The authors argue an SSM-based backbone with O(L) global context is a better fit.
2. **BERT-text conditioning (Calliffusion's recipe) is imprecise for calligrapher names.** BERT was trained on web text; it doesn't know that "曹全碑" is a 隸書 inscription vs the name of a real person. Token embeddings drift.
3. **Existing calligraphy datasets are tiny.** ZiGAN's 9 sub-datasets each have ~6k samples. Moyun ships **Mobao** — 1.93 M images, 2 681 calligraphers, 4 660 chars, 6 scripts — built by SAM + k-means binarization on a 1.2 M raw crawl.

The paper's central architectural idea is twofold:

- **Vision Mamba (Mamba2)** as backbone, operating on a 32×32×4 VAE latent.
- **TripleLabel** conditioning: three INDEPENDENT trainable embedding tables (calligrapher / font / character), summed into one vector, then projected to DiT-style scale-shift modulation parameters fed into each Mamba block.

The training objective is standard **DDPM denoising loss** with **classifier-free guidance** at inference.

## 2. Mamba math primer

The blind reimpl ships a pure-PyTorch S6 ("Mamba1") sequential scan in `src/moyun/mamba_block.py` because `mamba-ssm` requires CUDA + Triton and can't build on macOS (the Mac smoke-test machine) or unmodified Windows.

A linear SSM at time `t` evolves as

```
h_t   = A_bar_t * h_{t-1} + B_bar_t * x_t        (state update)
y_t   = C_t * h_t + D * x_t                      (output)
```

In **S6 / Mamba1** the matrices `Δ, B, C` are *functions of the input* (the "selective" part) — different tokens write/read different things in/out of the hidden state. Discretization uses

```
A_bar = exp(Δ * A)            (diagonal A, elementwise)
B_bar ≈ Δ * B                 (Euler approx; ZOH in the paper)
```

`A` is parameterized in log-space as a learnable `(d_model, d_state)` tensor with negative real eigenvalues (init `-1, -2, …, -d_state` per channel — a HiPPO-LegT-like init). This guarantees `A_bar = exp(Δ * A) ∈ (0, 1)` so the recurrence is stable.

`Δ` itself is a low-rank parametrization: project `x_t` to `(dt_pre, B, C)` with `nn.Linear`, then `dt = softplus(dt_proj(dt_pre))` (positive). `softplus` keeps the time-step strictly positive so `exp(Δ * A)` stays bounded.

The output `y_t` adds a residual `D * x_t` and is then **gated** by a SiLU-activated parallel branch (the Mamba "expand=2" idiom — split the input into `(x_for_ssm, z_for_gate)` via the 2× in-projection).

**Mamba2** (SSD) restricts `A` to a scalar per head so the recurrence becomes a structured matmul that can use tensor cores. The output is algorithmically identical to S6 when the rank is reduced; we keep the full S6 form here since the kernel-level perf gain is moot on CPU.

**Vision Mamba bidirectionality**: a single forward scan only sees past context. Following Zhu et al. 2024 we run a forward AND reversed scan and average; this restores the "full image context" property at O(L) total cost.

## 3. TripleLabel conditioning

Three INDEPENDENT embedding tables:

- `writer  : nn.Embedding(num_calligraphers + 1, hidden_dim)`
- `script  : nn.Embedding(num_scripts + 1, hidden_dim)`
- `char    : nn.Embedding(num_chars + 1, hidden_dim)`

The `+1` is for a reserved index 0 which is the `[NULL]` token (used for classifier-free guidance dropout). The `[NULL]` rows are **zero-initialized** so the uncond branch starts as a clean "no signal".

At training, with probability `cfg_drop_prob` (default 0.1), all three labels are jointly replaced by the `[NULL]` row for a given sample. The model learns to denoise both with and without conditioning, and at inference

```
ε_guided = ε_uncond + ω * (ε_cond - ε_uncond)
```

allows trading off diversity vs label adherence with `ω`.

A critical correctness property (DL review rubric, Moyun row): **the three tables must remain separate `nn.Embedding` modules with distinct learnable weights** — sharing weights would break the paper's promise of independent attribute control. `tests/test_smoke.py::test_triple_label_embeddings_are_distinct_modules` checks this explicitly.

## 4. Per-block modulation (DiT-style scale-shift)

The summed `e_total = e_writer + e_script + e_char ∈ ℝ^H` is added to a sin/cos+MLP time embedding `t_emb ∈ ℝ^H`, then projected:

```
cond     = e_total + t_emb                        # (B, H)
modulate = MLP(cond) ∈ ℝ^{4 * H * num_blocks}     # one matmul for all blocks
per_block = modulate.chunk(num_blocks, dim=-1)    # each (B, 4H)
```

Each block receives `(scale_ssm, shift_ssm, scale_ffn, shift_ffn)` (each `(B, H)`) and applies adaLN inside:

```
h_modulated = LN(x) * (1 + scale) + shift
```

before the SSM and FFN sublayers. The final linear of the modulation MLP is **zero-initialized** (DiT §3.2 "adaLN-Zero"); this means each block starts as an identity time-mixer and the network learns the modulation gradually. This is critical for stable training of deep DiT-style stacks.

## 5. Data flow

```
input image x_0 (B, 1, H, W) — grayscale glyph (or 4-ch VAE latent in production)
  -> q-sample with random t -> x_t (same shape)
  -> patchify (Conv2d kernel=patch_size stride=patch_size, hidden_dim out)
  -> + learnable positional embedding
  -> [VisionMambaBlock × N]   ← scale/shift from MLP(e_total + t_emb)
  -> LayerNorm
  -> patch-unembed (ConvTranspose2d kernel=patch_size stride=patch_size)
  -> ε_pred (same shape as x_t)

loss = MSE(ε_pred, ε_true)   # standard DDPM eps-prediction
```

We use `paper_reimpl_shared.diffusion.gaussian.GaussianDiffusion` for `q_sample`, `p_sample`, DDIM, and the CFG path. Moyun.forward takes the same kwarg signature as the other 8 papers' models (`x_t, timesteps, *, char_id, script_id, writer_id, content, ref_images, ref_valid, …`), but silently ignores `content` / `ref_images` / `ref_valid` since Moyun is id-conditioned, not image-conditioned.

## 7. What was deliberately NOT implemented

- **VAE encoder/decoder**: the paper assumes a pretrained VAE producing 32×32×4 latents. We do not ship one (no public Mobao-aware VAE exists). The default config defines `in_channels=4` for VAE-latent mode, and the smoke test runs with `in_channels=1` directly on grayscale pixels. Production training will need to plug in a VAE — either SD-1.5's (its latent priors are wrong for binary glyphs, expect degraded outputs) or a Stage-0 sub-training that fine-tunes a VAE on Ernantang/TTF renders.
- **SAM + k-means binarization pipeline**: Moyun's data preparation pipeline for raw web crawls. Ernantang data is already cleaned, so this is not needed for our use case.
- **`mamba-ssm` CUDA kernel**: replaced by pure-PyTorch scan (see `mamba_block.py` docstring for the swap point if a fast path becomes necessary later).
- **OCR-based structure metric**: paper §4.1 uses a Chinese OCR system to measure character recognizability — we defer this to Phase 3 eval.

## 8. Connection to the broader Ernantang baseline

TripleLabel maps directly onto Ernantang's metadata:
- `writer_id` → 24 calligraphers
- `script_id` → 楷/行/草/隸/篆 (5 classes)
- `char_id` → 4 659 chars

This is the cleanest paper-to-Ernantang mapping of any of the 8 papers we are reimplementing — there's no IDS dictionary, no stroke order, no 12-structure labels, no quadtree. The only data gap is **scale**: Mobao's 2 681 writers vs Ernantang's 24; full-scale paper-comparison is not feasible without Mobao access, but a scaled-down Ernantang Stage-C is well-defined.
